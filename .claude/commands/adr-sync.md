ADR 변경 후 연관 문서의 동기화 여부를 점검한다.

## 입력

`$ARGUMENTS` — ADR 식별자. 다음 형식을 모두 허용한다: `ADR-002`, `002`, `2`

## 실행 절차

### 1. ADR 파일 특정

`$ARGUMENTS`를 정규화한다.
- 숫자만 있으면 3자리로 패딩: `2` → `ADR-002`, `10` → `ADR-010`
- `ADR-` 접두사가 없으면 추가: `002` → `ADR-002`
- 결과 형식: `ADR-NNN`

`docs/adr/` 디렉토리에서 `ADR-NNN-*.md` 패턴 파일을 찾는다.
파일이 없으면 "ADR-NNN 파일을 찾을 수 없습니다. docs/adr/ 목록을 확인하세요." 출력 후 종료.

### 2. ADR 내용 읽기

대상 ADR 파일 전체를 읽는다. 다음 항목을 파악한다:
- **결정 내용**: 어떤 기술/패턴을 선택했는가
- **변경된 내용**: 기존 결정과 달라진 부분이 있는가 (파일이 방금 수정된 경우)
- **영향 범위**: 어떤 컴포넌트/레이어에 영향을 주는가

### 3. 의존 문서 스캔

아래 파일들에서 해당 ADR ID(`ADR-NNN`)를 참조하는 섹션을 찾는다:

| 파일 | 스캔 목적 |
|------|----------|
| `CLAUDE.md` | 구현 규칙, 핵심 패턴 설명 |
| `docs/ARCHITECTURE.md` | 패키지 구조, 인터페이스, DB 스키마 |
| `DECISIONS.md` | 설계 쟁점 및 선택 근거 요약 |
| `docs/adr/DECISIONS.md` | ADR 인덱스 및 결정 흐름 |
| `docs/adr/ADR-*.md` (다른 ADR) | cross-reference 또는 근거로 인용한 경우 |

각 파일을 읽고 ADR ID가 언급된 섹션(헤더 포함 전후 문맥)을 수집한다.

### 4. 영향 분석 및 보고

수집한 섹션을 ADR의 현재 내용과 비교한다. 다음 테이블 형식으로 출력한다:

```
## ADR-NNN 동기화 점검 결과

### ADR 핵심 변경 요약
(ADR 파일에서 파악한 결정 내용 2-3줄)

### 연관 문서 영향 분석

| 파일 | 섹션 | 상태 | 업데이트 제안 |
|------|------|------|--------------|
| CLAUDE.md | ## 섹션명 | 🔴 업데이트 필요 / 🟡 확인 권장 / 🟢 일치 | 구체적 변경 내용 |
| docs/ARCHITECTURE.md | ### 섹션명 | ... | ... |
| DECISIONS.md | ## 섹션명 | ... | ... |

### 권장 조치
1. (우선순위 높은 항목부터 구체적 수정 지시)
```

**상태 기준**:
- 🔴 업데이트 필요: 문서 내용이 ADR 결정과 명확히 불일치
- 🟡 확인 권장: ADR 변경에 따라 연관 내용이 있으나 충돌 여부 불명확
- 🟢 일치: 문서가 ADR을 정확히 반영 중

참조가 없는 파일은 테이블에서 생략한다.
